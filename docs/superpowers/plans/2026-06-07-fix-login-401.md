# Fix Login 401 Error Interceptor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent the axios interceptor from attempting a token refresh when a login request fails with a 401 error.

**Architecture:** We will add a short-circuit condition in the response interceptor in `src/api/client.ts`. When `originalRequest.url?.endsWith('/auth/login')` and the error status is 401, we will reject the error immediately, bypassing the refresh logic.

**Tech Stack:** Axios, TypeScript, React

### Task 1: Update API Client Interceptor

**Files:**
- Modify: `src/api/client.ts:69-78`

- [ ] **Step 1: Add login short-circuit to response interceptor**

Modify `src/api/client.ts` to add the short-circuit for `/auth/login`:

```typescript
    // 1. Short-circuit: if the refresh request itself fails with a 401
    if (
      error.response?.status === 401 &&
      originalRequest.url?.endsWith('/auth/refresh')
    ) {
      await handleTokenRevocation();
      return Promise.reject(error);
    }

    // 2. Short-circuit: if the login request fails with a 401, do not attempt to refresh
    if (
      error.response?.status === 401 &&
      originalRequest.url?.endsWith('/auth/login')
    ) {
      return Promise.reject(error);
    }
```
