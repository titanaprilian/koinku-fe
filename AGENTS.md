# tanstack-auth-starter

React + Vite SPA with JWT auth, TanStack Router, TanStack Query, and shadcn/ui.

## Commands

- `bun install` ŌĆö install dependencies (project uses bun lockfile)
- `bun run dev` ŌĆö start Vite dev server on http://localhost:5173
- `bun run build` ŌĆö type-check with `tsc -b` then bundle with Vite
- `bun run lint` ŌĆö ESLint across all `.ts`/`.tsx` files
- `bun run preview` ŌĆö serve production build locally

## Gotchas

- **Do NOT edit `src/routeTree.gen.ts`** ŌĆö this file is auto-generated by `@tanstack/router-plugin/vite`. Add or rename files in `src/routes/` and the plugin regenerates it on dev server restart.
- **Access tokens live in memory only.** Never store tokens in localStorage or sessionStorage. The singleton `authService` in `src/features/auth/auth-service.ts` holds the access token in a private class field. Session restore on page reload uses the HttpOnly refresh cookie via `POST /auth/refresh`.
- **Axios interceptors handle 401 retry automatically.** Do not add manual refresh logic in feature code. The response interceptor in `src/api/client.ts` queues concurrent requests during a refresh and replays them. If the refresh itself 401s, it clears the session and redirects to `/login`.
- **Protected routes use a layout route, not per-route guards.** Place new authenticated pages inside `src/routes/_authenticated/`. The `beforeLoad` guard in `src/routes/_authenticated.tsx` handles the redirect. Do not duplicate auth checks in child routes.
- **Form validation uses TypeBox schemas, not Zod.** This project uses `@sinclair/typebox` with `@hookform/resolvers/typebox`. Do not import `zod` or `@hookform/resolvers/zod`.
- **UI components come from shadcn/ui (Base Nova style).** Add new components with `bunx shadcn@latest add <component>`. Do not create custom primitives that duplicate existing shadcn/ui components in `src/components/ui/`.
- **`VITE_API_URL` defaults to `http://localhost:4000` when unset.** The fallback is hardcoded in `src/api/client.ts`. For production, set this env var at build time ŌĆö Vite statically replaces `import.meta.env.*` values.

## Conventions

- **Feature-based architecture:** each domain gets a folder under `src/features/<name>/` with co-located `api.ts`, `types.ts`, `components/`, and `hooks/`. Do not put feature-specific code in top-level `src/components/` or `src/hooks/`.
- **Path alias:** use `@/` for imports from `src/` (configured in `tsconfig.app.json` and `vite.config.ts`). Never use relative paths that climb more than one level (`../../`).
- **Router context carries auth state:** the root route (`src/routes/__root.tsx`) defines `RouterContext` with `queryClient` and `auth`. Access auth state via `context.auth` in `beforeLoad`, not by importing `authService` directly in route files.
- **Styling:** Tailwind CSS v4 with CSS variables defined in `src/index.css`. Use `shadcn/tailwind.css` tokens ŌĆö do not introduce a separate color palette.

## Do not commit

- `node_modules/`
- `dist/`
- `.env.local` (contains `VITE_API_URL` and potential secrets)
- `src/routeTree.gen.ts` changes made by hand (they will be overwritten)
